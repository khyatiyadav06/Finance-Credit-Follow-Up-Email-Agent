Tech Stack

Decisions & Rationale

🤖 LLM: Claude Sonnet 4

Why: Superior instruction-following for strict JSON output. Better tone calibration for professional communication vs GPT-4o. Anthropic's Constitutional AI reduces hallucination risk in financial contexts. Sonnet provides 95% of Opus quality at 5× lower cost.

🔗 Framework: LangGraph

Why: Explicit, auditable state transitions — critical for financial workflows. Every step is traceable. LangChain's ReAct loop is less predictable in production. LangGraph's graph model makes the audit story clear.

✅ Validation: Pydantic v2

Why: Structured output from day one eliminates parsing bugs. All inputs and LLM outputs validated before use. Validators sanitize fields against prompt injection.

📊 UI: Streamlit

Why: Fastest path to a working dashboard. Shows invoice queue, audit log, email previews, and single-invoice run — all in one interface. Production could upgrade to FastAPI + React.

Security

6 Risk Mitigations Implemented

🛡️ Prompt Injection

client_name sanitized via regex — strips all non-alphanumeric chars. Raw CSV never touches LLM directly — must pass Pydantic validation first.

🔏 Data Privacy / PII

Audit log stores 200-char body preview only — never the full email. .env excluded from Git. PII processed locally; only structured fields sent to API.

🔑 API Key Exposure

All credentials via python-dotenv only. .env in .gitignore. .env.example provided. Runtime check fails fast if key is missing.

🤔 Hallucination Risk

Structured JSON output mode. Post-generation check verifies invoice_no appears in email. Human-in-the-loop: DRY_RUN=true by default.

🚫 Unauthorised Access

CLI requires explicit --live flag. Dashboard runs locally only. Production: OAuth / API key on any exposed endpoints.

📬 Email Spoofing

Sender email from env vars only — never derived from invoice data. SPF/DKIM/DMARC setup documented in README. Dry-run default prevents test accidents.

Learnings & Next Steps

Key Takeaways

🏗️

Structure first, intelligence second

Starting with Pydantic models before writing any LLM code saved hours of debugging parsing errors mid-project.

🔐

Security is architecture, not an afterthought

Prompt injection protection required rethinking data flow entirely — validators must run before any data reaches the LLM.

🧪

Dry-run mode is essential for financial agents

Testing with real email sends would have cost actual client communications. Default DRY_RUN=true is non-negotiable.
